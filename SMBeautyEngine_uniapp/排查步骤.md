# 排查"无法运行到手机或模拟器"问题

## 查看错误信息的方法

### 方法一：HBuilderX 控制台
1. **显示控制台**：
   - 菜单：**视图** → **显示控制台** 或 **显示输出面板**
   - 或查看 HBuilderX 底部是否有标签页（控制台/输出/终端等）
   - 或快捷键：`Cmd + J` (Mac) 或 `Ctrl + J` (Windows)

2. **运行时的信息**：
   - 点击 **运行** → **运行到手机或模拟器**
   - 查看弹出的提示框或底部状态栏的错误信息

### 方法二：项目属性检查
1. 右键项目根目录（`SMBeautyEngine_uniapp`）
2. 选择 **属性** 或 **项目设置**
3. 查看项目类型是否为 "uni-app"

### 方法三：重新创建项目（推荐）
如果 HBuilderX 无法识别项目类型，建议重新创建：

1. **创建新项目**：
   - 文件 → 新建 → 项目
   - 选择 **uni-app** → **默认模板**
   - 项目名称：`pixelfree-demo`
   - 位置：选择其他目录（不要覆盖当前项目）

2. **复制文件到新项目**：
   ```
   从 SMBeautyEngine_uniapp 复制到新项目：
   - pages/ 目录
   - js_sdk/ 目录  
   - static/ 目录
   - nativeplugins/ 目录
   - src/ 目录
   - manifest.json（覆盖）
   - pages.json（覆盖）
   - main.js（覆盖）
   - App.vue（覆盖）
   ```

3. **在新项目中运行**
